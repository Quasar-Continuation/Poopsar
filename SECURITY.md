# Security Policy

## Supported Versions

We release patches for security vulnerabilities for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 2.x.x   | :white_check_mark: |
| 1.4.x   | :x:                |
| < 1.4   | :x:                |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

If you discover a security vulnerability within Pulsar, please follow these steps:

1. **Contact Us Privately**: Send details to our [Telegram channel](https://t.me/novashadowisgay) with the subject "SECURITY"
2. **Provide Details**: Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

### What to Expect

- **Acknowledgment**: We will acknowledge receipt of your vulnerability report within 48 hours
- **Updates**: We will provide regular updates on our progress
- **Disclosure**: We follow responsible disclosure practices and will work with you to determine an appropriate disclosure timeline
- **Credit**: If you wish, we will credit you in our security advisory

## Security Best Practices for Users

When using Pulsar:

1. **Always use the latest version** - Security patches are only applied to supported versions
2. **Use strong encryption** - Ensure TLS is properly configured
3. **Limit access** - Only allow connections from trusted IP addresses
4. **Monitor logs** - Regularly review logs for suspicious activity
5. **Legal compliance** - Only use Pulsar on systems you own or have explicit permission to access

## Security Features

Pulsar includes several security features:

- **TLS 1.2+ Encryption** - All communication is encrypted
- **Protocol Buffers** - Efficient and secure serialization
- **Anti-VM/Anti-Debug** - Protection against analysis environments
- **Built-in Obfuscation** - Protect your builds from reverse engineering

## Disclaimer

Pulsar is a powerful remote administration tool. Users are responsible for:

- Ensuring legal compliance in their jurisdiction
- Obtaining proper authorization before accessing systems
- Using the tool ethically and responsibly

The developers are not responsible for misuse of this software.
